# Лицензии сторонних компонентов

`PhotoArchive.exe` — это `photosort_win.py` (Apache License 2.0, см.
[`LICENSE`](./LICENSE)/[`NOTICE`](./NOTICE)), собранный PyInstaller'ом ВМЕСТЕ с четырьмя внешними бинарниками из
`bin/` (см. `bin/README-BIN` в исходном репозитории -- файл для сборки из исходников, не
входит в поставку `.exe`). Ни один из них не написан в этом
проекте и не лицензирован так же, как собственный код, — при публикации `.exe` как GitHub
Release нужно соблюдать условия каждого из них независимо от того, под какой лицензией сам
`photosort_win.py`. Этот файл — не юридическая консультация, а практическая памятка «что
приложить и почему», написанная по итогам аудита безопасности/надёжности 2026-07-10.

Версии сознательно не зафиксированы в `bin/README-BIN` (в исходном репозитории, см. этот файл — берутся
актуальные официальные сборки на момент сборки `.exe`), поэтому вместо того чтобы держать в
репозитории текст лицензии конкретной версии (который может разойтись с тем, что реально
скачал сборщик), процедура такая:

**При сборке — вместе с самим бинарником сохраняйте LICENSE/COPYING-файл, который идёт в
официальном архиве, в `bin/licenses/<имя>/`** (создаётся сборщиком вручную, содержимое не
проверяется программой, только копируется). `build\build.bat` копирует всю папку
`bin/licenses/` в `build\dist\licenses\` рядом с собранным `PhotoArchive.exe` — так в
GitHub Release попадает точный текст лицензии именно той версии, что реально была
скачана, а не устаревшая копия из репозитория.

```
bin/
  licenses/
    exiftool/   <- LICENSE (Perl Artistic/GPL) из архива exiftool
    ffmpeg/     <- COPYING.LGPLv2.1 (или GPL, если сборка GPL -- см. предупреждение ниже)
    7zip/       <- License.txt из инсталлятора/архива 7-Zip
    unrar/      <- license.txt из архива UnRAR
```

Если `bin/licenses/` не заполнена — `build.bat` печатает предупреждение (как и для
недостающих самих бинарников), сборка не блокируется, но перед публикацией релиза
`bin/licenses/` должна быть на месте (см. чек-лист выпуска в исходном репозитории).

## Сводка по компонентам

### ExifTool (`exiftool.exe` + `exiftool_files\`)

- Автор: Phil Harvey, [exiftool.org](https://exiftool.org/).
- Лицензия: та же, что и у Perl — на выбор пользователя Artistic License 1.0 ИЛИ GNU GPL
  (dual license). `exiftool_files\` содержит встроенный портативный Perl-интерпретатор —
  та же схема лицензирования распространяется и на него.
- Требование при распространении: сохранить уведомление об авторстве/лицензии
  (`bin/licenses/exiftool/`), исходный код доступен на exiftool.org — отдельно
  прикладывать не нужно (не модифицируется).

### FFmpeg / FFprobe (`ffmpeg.exe`, `ffprobe.exe`)

- [ffmpeg.org](https://ffmpeg.org/) (или сторонняя официальная Windows-сборка, см.
  `bin/README-BIN` (в исходном репозитории)).
- Лицензия зависит от НАБОРА кодеков, включённых в конкретную сборку:
  - **LGPL v2.1+** — если сборка собрана БЕЗ `--enable-gpl` (не включает GPL-only кодеки
    вроде x264/x265). Обязательство сводится к сохранению текста LGPL и уведомления, что
    исходный код доступен на ffmpeg.org — не требует раскрытия исходного кода
    `photosort_win.py`.
  - **GPL v2+/v3** — типичные готовые сборки "full"/"essentials" с ffmpeg.org-зеркал (в т.ч.
    исторически использовавшаяся в этом проекте gyan.dev-сборка, см. `bin/README-BIN` (в исходном репозитории))
    собраны С GPL-компонентами. GPL требует явного предложения исходного кода (source offer)
    для GPL-компонента при распространении бинарника, и потенциально накладывает copyleft
    на комбинацию (юридически спорно для `subprocess`-вызова внешнего `.exe`, а не линковки
    в процесс — но это не позиция проекта, а факт неопределённости, которую стоит избегать).
  - **РЕКОМЕНДАЦИЯ проекта: брать LGPL-only сборку** (например тег `lgpl` у
    [BtbN/FFmpeg-Builds](https://github.com/BtbN/FFmpeg-Builds/releases) — есть отдельные
    релизы `*-lgpl-*.zip` и `*-gpl-*.zip`, брать именно `lgpl`) — см. обновлённую инструкцию
    в `bin/README-BIN` (в исходном репозитории). Если по какой-то причине собрана GPL-сборка — это осознанное
    исключение, тогда обязательно приложить source-offer (ссылку на исходный код именно той
    версии ffmpeg, из которой собран бинарник) в `bin/licenses/ffmpeg/`, помимо самого текста
    лицензии.

### 7-Zip (`7z.exe` + `7z.dll`)

- [7-zip.org](https://www.7-zip.org/), автор Igor Pavlov.
- Лицензия: GNU LGPL для большей части кода; часть, отвечающая за распаковку RAR
  (совместимость), — под отдельным ограничением "unRAR license" (нельзя использовать этот
  код для разработки RAR-АРХИВАТОРА — программа его только распаковывает через `7z.exe` как
  внешний процесс, использования кода как такового нет).
- Требование: сохранить `License.txt` из архива/инсталлятора (`bin/licenses/7zip/`).

### UnRAR (`UnRAR.exe`)

- [rarlab.com](https://www.rarlab.com/rar_add.htm), автор Alexander Roshal.
- Лицензия: собственная (freeware) — бесплатное распространение НЕИЗМЕНЁННОГО `UnRAR.exe`
  разрешено при условии сохранения текста лицензии и явного указания, что это компонент
  RARLAB, отдельно как «архиватор» не продаётся. Точный текст — `license.txt`, идущий в
  официальном архиве UnRAR (сохранить как есть в `bin/licenses/unrar/`).

## Что проверить перед публикацией релиза

См. чек-лист выпуска в исходном репозитории (раздел про `bin/licenses/`) — вкратце: `bin/licenses/`
заполнена всеми четырьмя папками, `build\dist\licenses\` появилась после сборки, и она
приложена к GitHub Release вместе с `PhotoArchive.exe` (как отдельный zip или как есть).
